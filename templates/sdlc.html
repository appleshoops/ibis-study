Software Development Life Cycle (SDLC)

Requirements Definition

The objective of this app is to allow students to enhance their studies by creating flashcards to help memorise things for tests, and to create and use quizzes to help consolidate their knowledge.

The functional requirements include:

  • User authentication through SQLite 3 and Werkzeug
  • Database setup with SQLite for Users, Quizzes, and Progress Logs
  • Ability to log progress for development process through a form within the website
  • Basic quiz interface that can display flashcards and multiple choice questions
  • Documentation of the SDLC

The non-functional requirements include:

  • The app must be responsive
  • The app must use Flask for backend routing and Jinja for templating
  • Insecure and secure practices both need to be displayed
  • The code must be version controlled with Git
  • The app should handle basic errors using the Flask flash system

The constraints include:

  • Use only Flask, SQLite 3, and built in Python libraries such as Werkzeug for password hashing
  • Quizzes should be limited to 1 subject in this prototype
  • No PWA features such as voice/ML as they will be added later

My acceptance criteria is:

  • Users can login using username and password within 1 second
  • A user can take a quiz and have it marked within 2 seconds
  • The system prevents users from deleting flashcard sets or quizzes made by other users
  • The system prevents users from creating accounts with emails that have already been used to make an account
  • All features work on the latest versions of all major browsers

Determining Specifications

The functional outcomes are:

  • The login and register functions take text input from the user and insert them into the SQL table to verify their identity
  • The quiz system allows users to name a quiz and input a number of questions as an integer, then define 4 multiple choice options per question as well as a correct answer
    • During quizzes, the user can select 1 of 4 boxes per question and once submitted, they will be graded
  • The flashcard system allows users to name a flashcard set and input the number of flashcards within it, then define a front and back side for each flashcard
    • When studying the flashcards, the user can click on it to flip it around and view the other side
  • The progress logs feature allows users to create individual progress logs by providing a date through a calendar picker, a title, and a description through a text area.

The non-functional outcomes are:

  • There will be a navigation bar at the top of the screen to easily move between major pages
  • Related elements will be grouped together with divs
  • The accent colour will be a light green, with a light red to indicate significant actions or errors
  • Major elements will be placed in the center of the screen
  • The wording of the messages will be professional yet friendly
  • The fonts used will be consistent and fit the professional and bold aesthetic of the website

Design

The key aspects are:

  • Recognising the need to help students with their studying by providing easy access to quiz and flashcard set creation
  • The quiz function will compare the user's input to what the correct answer is defined as in the SQL Questions table and will give points accordingly for each question
  • HTML, CSS, SQL, and Python will be used with Flask and Jinja
  • Future options to make it fun could be adding a leaderboard for points
